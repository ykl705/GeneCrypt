## 一、技能基因（主动技能）

所有主动技能基因均为 **隐性**（`dominant: False`），即纯合时才会表达技能。
每条染色体上基因按顺序排列，紧密相邻（无间隔）。

## 三、敌方被动技能（ENEMY_PASSIVES）

敌方单位从31关开始可能携带被动能力，通过`passive_abilities`字段配置。
所有被动效果在战斗中由`BattleSystem.process_enemy_passives()`每12帧处理一次。

### 光环类被动
| 被动ID | 名称 | 效果 |
|--------|------|------|
| `attack_aura` | 攻击灵气 | 每回合提升全体友方20%攻击力 |
| `defense_aura` | 防御灵气 | 每回合提升全体友方30%防御力 |
| `speed_aura` | 速度灵气 | 每回合提升全体友方20%速度 |
| `heal_aura` | 治愈灵气 | 每回合恢复全体友方3%最大生命值 |
| `shield_aura` | 护盾灵气 | 每回合为全体友方提供5%最大生命值的护盾 |
| `toxic_aura` | 毒瘴灵气 | 每回合对全体敌方施加中毒效果(30伤害, 2回合) |
| `thorns_aura` | 荆棘灵气 | 反弹25%受到的伤害给攻击者 |

### 入场被动
| 被动ID | 名称 | 效果 |
|--------|------|------|
| `fortify` | 钢铁意志 | 登场时提升全体友方25%最大生命值(仅触发一次) |

### 亡语被动 (死亡时触发)
| 被动ID | 名称 | 效果 |
|--------|------|------|
| `rage` | 怒火 | 死亡时提升全体友方30%攻击力，持续5回合 |
| `vengeance` | 复仇 | 死亡时对全体敌方造成50%最大生命值的伤害 |

### 持续被动
| 被动ID | 名称 | 效果 |
|--------|------|------|
| `regeneration` | 再生 | 每回合恢复自身8%最大生命值 |
| `bless` | 净化祝福 | 每回合移除全体友方一个负面效果 |
| `berserk` | 狂怒 | 生命低于50%时攻击力翻倍 |

### 携带被动的敌方单位
- **指挥官**: `attack_aura` + `defense_aura`
- **战鼓手**: `speed_aura` + `heal_aura`
- **守护之灵**: `fortify` + `shield_aura`
- **复仇之魂**: `rage` + `vengeance`
- **腐蚀之源**: `toxic_aura` + `bless`
- **湮灭体**: `vengeance` (亡语复仇)
- **铁甲堡垒**: `thorns_aura` (荆棘反伤)
- **治疗先驱**: `regeneration` (自我再生)

### 处理时机
```
每12帧 → update_status() → update_poison_damage() → process_purify() 
       → process_enemy_passives() → cleanup_summons()
```

## 四、游戏机制说明

### 技能触发概率
- 战斗中每回合有 **50%** 概率使用技能（vs 普通攻击）
- 从拥有技能中**随机选择一个**

### 技能强化科技
- 技能伤害/治疗/护盾受科技树「技能强化」等级影响
- 每级 +10% 效果：`mult = 1 + 等级 × 0.1`

### 伤害计算流程
1. 先计算技能基础值（base_damage + 攻击力 × 系数）
2. 应用科技强化倍率
3. 目标`take_damage()`中：先扣护盾 → 再扣防御 → 最低1伤害
4. 触发生命值归零判定
5. 触发荆棘反伤

### 被动技能添加方法
1. 在 `GENE_TEMPLATES` 中添加新条目，`category: 'passive'`
2. 设置 `passive_name`（技能显示名称）
3. 定义B序列（激活序列前缀）和检测逻辑
4. 在 `BattleCard.take_damage()` 中添加触发逻辑
